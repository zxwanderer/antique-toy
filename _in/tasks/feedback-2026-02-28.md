## 5. FEEDBACK — 2026-03-01 (Sergio Morales via Telegram)

### BUG-6: Ch02 — pixel_addr uses RRA instead of RLA for L byte calculation
- **Chapter:** Ch02 (Screen as Puzzle)
- **Files:** `chapters/ch02-screen-as-puzzle/draft.md` (line 143), `examples/pixel_demo.a80` (line 118)
- **Description:** The second block of `pixel_addr` shifts bits 5-3 of y (character row within third) into L[7:5] for the screen address. The original code used `RRA × 3` (right rotate through carry), which shifts those bits to positions [2:0] — then `AND $E0` zeroes them out. Result: L always gets $00 for the character row component, causing all pixels to collapse into the first character row of each third.
- **Fix:** Replace `RRA / RRA / RRA` with `RLA / RLA` (left rotate by 2). This puts y[5:3] into L[7:5] correctly. Also saves 1 byte and 4 T-states (87T vs 91T).
- **Source:** Sergio Morales (Telegram, 2026-03-01)
- **Priority:** P1
- **Resolution:** FIXED in v22. Both draft.md and pixel_demo.a80 updated. Screenshot regenerated.

## 7. FEEDBACK — 2026-03-01 (Ped7g via Discord)

### IMP-4: Ch04 — `mul_signed` uses outdated 2×abs approach
- **Chapter:** Ch04 (Maths)
- **File:** `chapters/ch04-maths/draft.md`, lines 143-205
- **Description:** Ped7g notes that the current `mul_signed` uses the "2×abs + final neg" pattern, which is not state-of-the-art. The modern approach: perform unsigned multiply, then fix the upper byte by subtracting the other operand if a sign bit is set. Mathematical identity: `signed_result = unsigned_result - 256*(sign(a) ? b : 0) - 256*(sign(b) ? a : 0)`. This avoids abs/neg entirely. On Z80N with MUL instruction, this yields ~70T (vs ~250T). On pure Z80, savings are modest but the technique is cleaner. Ped7g's Z80N implementation: https://github.com/ped7g/ZXSpectrumNextMisc/blob/master/snippets/mul.i.asm#L218. He mentions base/busy from SinDiKat may have pure Z80 variants.
- **Fix:** Add a sidebar explaining the fix-unsigned technique with the mathematical identity. Keep existing mul_signed (it works, it's pedagogical), but show the better approach as a "Going Further" note.
- **Source:** Ped7g (Discord, 2026-03-01)
- **Priority:** P2
- **Resolution:** FIXED in v22. Added "Going Further: Fix-Unsigned Technique" sidebar to Ch04.
